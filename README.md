# QG-AI使用与个人贡献说明
QG后台学习



- #### AI制作前端

  - 配置好node.js，把admin和student的controller发给AI，AI便哐哐哐一路把.ts都做好了。至于页面展示美观我没做任何要求，仅仅让AI自行发挥，结果还不错，干净整洁，报错用了不同颜色。

- #### AI查找错误

  - 在启动springboot的时候疯狂报错，在终端clean install的时候又疯狂报错，AI也找不到什么错误，只是一味地让我降低spring和mybatis-plus的版本，原本的4.0.4的spring硬生生给我降到2.7，总算是解决了启动问题。后来问了一个大哥，他说mbplus对springboot3有专属的框架，应该不改版本，改依赖才是，可见AI有时不如人(或者deepseek不太行)。
  - 在启动成功后，注册登录学生时，一路无阻，直到在注册登录老师时出现bug，数据库记录的工号总是少了两个0。经AI检查，发现我工号用的是Long，会自动去除前导零。为了不大改，最后选择了在调用数据库的时候，先加两个零再与request.getId()进行对比。

- #### AI梳理

  - 实际上我一开始，完全不懂前后端直接怎么联系，什么是axios等。所有这方面的疑问，基本是让AI给我画图和举通俗例子给解决的。因为我觉得听课效率有点低，直接找AI对症下药。
